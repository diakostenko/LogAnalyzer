# Log Analyzer

Веб-инструмент для анализа nginx/apache access-логов с автоматической детекцией аномалий.

Выполнен студенткой 2 курса направления 09.03.03 «Прикладная информатика» в рамках летней производственной практики.

---

## Что делает

Загружаете лог-файл — приложение парсит его на сервере и выдаёт:

- **График трафика** — запросы по часам с подсветкой пиков
- **Распределение статус-кодов** — 2xx / 3xx / 4xx / 5xx
- **Топ IP-адресов и эндпоинтов** — по количеству запросов, с флагами подозрительности
- **Детекция аномалий** — автоматически находит брутфорс, пики трафика, сканеры уязвимостей и всплески серверных ошибок
- **Таблица сырых записей** — страницы с поиском и фильтрами
- **Экспорт** — скачивание результат анализа в JSON или TXT

---

## Логика детекции аномалий

Анализатор выполняет четыре независимые проверки:

| Тип | Условие срабатывания |
|---|---|
| Пик трафика | Одна минута превышает среднее значение в 5 и более раз |
| Брутфорс | IP с 100+ запросами, 70%+ из которых 4xx, сконцентрированных на одном эндпоинте |
| Сканер | IP с 30+ запросами, 5+ уникальными путями, 50%+ из которых 404 |
| Ошибки сервера | 50+ ответов 5xx, составляющих 2%+ от общего трафика |

## Запуск

**Через Docker:**

```bash
git clone https://github.com/diakostenko/LogAnalyzer.git
cd LogAnalyzer
cp .env.example .env
docker-compose up --build
```

Открыть [http://localhost:3000](http://localhost:3000).

**Без Docker:**

```bash
npm install
cp .env.example .env
# убедитесь, что MongoDB запущена локально
npm run dev
```

## API

| Метод | Эндпоинт | Описание |
|---|---|---|
| `POST` | `/api/logs/upload` | Загрузить и проанализировать лог-файл |
| `GET` | `/api/logs` | Список всех анализов |
| `GET` | `/api/logs/:id` | Полный результат анализа |
| `GET` | `/api/logs/:id/export?format=json\|txt` | Скачать отчёт |
| `DELETE` | `/api/logs/:id` | Удалить анализ |
